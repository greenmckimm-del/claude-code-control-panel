04
The secret layer

Every session starts with a full briefing.

Claude Code reads a file called CLAUDE.md before every session. That file says who I am, what I'm building, and how I like things done. No re-explaining. No lost context. Memory files go further — they carry decisions across sessions, tools, and projects.

11
The Telstra angle

Pick one loop. Orchestrate it. See what happens.

Every one of these projects started with a single loop. One repetitive task that burned hours. I orchestrated it. Then I added another. You do not need twenty tools to start. You need one loop, one model, and one hour to set it up.

Weekly exec report
Now: 2 hours pulling from 5 dashboards and writing a narrative.
Orchestrated: scheduled agent runs Sunday night, draft lands in your inbox Monday morning, you review and send.
Customer complaint triage
Now: team wakes up to a wall of tickets, no prioritisation.
Orchestrated: first-pass categorisation and draft responses run overnight, team starts with a prioritised queue.
Board deck first draft
Now: 3 days pulling KPIs, strategy docs, and last quarter's deck.
Orchestrated: Claude reads all three, returns a structured first draft you sharpen rather than start from zero.

These examples use only information you already have access to. Proper data governance applies. Talk to Ben about guardrails before you connect anything to live Telstra data.
